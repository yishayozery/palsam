# 📦 קליטת מלאי קיים לפלוגה

## מתי להפעיל
- המשתמש מעלה צילומים של "שובר השאלה לכלי" / "ציוד אישי מפקדת X"
- מבקש לטעון את הציוד במערכת לפי פלוגה
- לרוב מבקש למחוק את המלאי הקיים בפלוגה ולהתחיל מחדש

## מבנה הצילום
טבלה עם עמודות (RTL):
- **מק"ט** (מספר)
- **תיאור מק"ט** (שם הפריט)
- **יח"ש** (יחידת שיווק - בד"כ "יח" או "ערכ")
- **תקן מוקצה** (סטנדרט מאושר - **לא** טוענים)
- **מלאי מוקצה** (כמות בפועל - **כן** טוענים)
- **פער** (הפרש)
- **סימול משלים** / **מנת אב** / **מיקום** / **מלאי חוץ** / **הערה**

⚠️ העמודה לטעינה היא **"מלאי מוקצה"** (הכמות בפועל), לא "תקן מוקצה".

## שאלות חובה לפני הטעינה
1. שם הפלוגה המדויק במערכת
2. אישור על מחיקת המלאי הקיים
3. קטגוריה ומחסן (בד"כ "ציוד" → `EQUIPMENT`)
4. שם המקבל / מי מסר (לרוב "מלאי קיים" אם זה ציוד קיים)

## זרימת ההפעלה

### שלב 1 - איסוף הנתונים
מתוך הצילומים, חלץ לכל פריט:
- `sku` - מק"ט (מספר)
- `name` - תיאור מק"ט
- `quantity` - **מלאי מוקצה** (לא תקן)

### שלב 2 - יצירת קובץ JSON
שמור ב-`data/company-stock/<company-slug>.json`:

```json
{
  "battalionCode": "CARMELI",
  "companyName": "מפקדה/אגם",
  "recipientName": "מלאי קיים",
  "reason": "טעינת מלאי קיים משובר השאלה (תאריך)",
  "category": "ציוד",
  "warehouseType": "EQUIPMENT",
  "defaultStatusName": "תקין",
  "items": [
    { "sku": "...", "name": "...", "quantity": N },
    ...
  ]
}
```

### שלב 3 - הרצה
```bash
npx tsx scripts/load-company-stock.ts data/company-stock/<file>.json
```

הסקריפט יבצע (ב-transaction):
1. ימצא גדוד+פלוגה
2. ימחק `StockBalance` של הפלוגה הזו (לא נוגע בקטלוג)
3. ימצא/יצור קטגוריה
4. ימצא סטטוס "תקין"
5. לכל פריט: `upsert` ItemType (לפי `battalionId + sku`)
6. יוצר תעודת `INTAKE` (status=COMPLETED) עם כל השורות
7. יוצר `StockBalance` במחזיק הפלוגה

### שלב 4 - אימות
```sql
SELECT i.sku, i.name, sb.quantity
FROM "StockBalance" sb
JOIN "ItemType" i ON sb."itemTypeId" = i.id
WHERE sb."holderId" = '<company-id>'
ORDER BY i.name;
```

או דרך ה-UI: `/stock` → בורר פלוגה → הפלוגה הספציפית.

## נקודות חשובות

- **כל פריט הוא QUANTITY** (לא סריאלי/אצווה) אלא אם המשתמש מציין אחרת.
- **התעודה נוצרת בשם המפ"מ הראשון** הפעיל בגדוד (כי הסקריפט רץ ללא session).
- **תעודה COMPLETED** — אין צורך באישור (מדובר על ציוד שכבר קיים).
- **אם פריט קיים כבר** (לפי SKU) — נעדכן את השם והקטגוריה אבל לא נכפיל.
- **`externalContact = recipientName`** ולא מ.א. (זה רישום של ציוד קיים, לא מסירה אמיתית).

## דוגמת קלט מהמשתמש שצריך לפרק לפריטים

המשתמש בד"כ ישלח שתי תמונות (עמוד 1+2 או 1+2+3). חלץ מכל עמוד את:
- מק"ט (העמודה הימנית ביותר, מספר 9-12 ספרות)
- תיאור (העמודה הסמוכה)
- מלאי מוקצה (העמודה החמישית-שישית מימין)

דוגמה לשורה אופיינית:
```
113516500 | מחסנית מתכת 30 כדור 16mm | יח | 2 | 66 | -64 | ...
```
→ `{ "sku": "113516500", "name": "מחסנית מתכת 30 כדור 16mm", "quantity": 66 }`

## טיפוסי פערים אפשריים
- **מק"ט קיים בקטלוג עם שם שונה** → הסקריפט יעדכן את השם לפי הקובץ החדש.
- **אותו מק"ט מופיע פעמיים** בצילום → תאחד לפני הקלט, או הסקריפט יזרוק שגיאה ב-create.
- **פלוגה לא קיימת** → השאלה את המשתמש איזה שם מדויק להשתמש; אל תיצור פלוגה חדשה אוטומטית.
- **חסר עמוד** (1/3 או 2/3) → השאלה את המשתמש אם להמשיך עם מה שיש.

## פלוגות קיימות בגדוד הנוכחי
לפני שמשתמש בשם פלוגה, הרץ בדיקה מהירה:
```ts
await prisma.holder.findMany({ where: { battalionId, kind: "COMPANY", active: true }, select: { name: true } });
```
